class Fred2.Core.Base.MetadataLogger[source]¶

Bases: object

This class provides a simple interface for assigning additional metadata to any object in our data model. Examples: storing ANNOVAR columns like depth, base count, dbSNP id, quality information for variants, prediction scores for peptides etc. Normally used by custom toolbox functions and importers.

The saved values are accessed via log_metadata() and get_metadata()

class Fred2.Core.Variant.Variant(id, type, chrom, genomePos, ref, obs, coding, isHomozygous, isSynonymous, experimentalDesign=None, metadata=None)[source]¶

Bases: Fred2.Core.Base.MetadataLogger

A Variant contains information about a single genetic modification of the reference genome.

Parameters:
  • id (str) – Variant id
  • type (Enum.VariationType) – An Enum type of the variant either SNP, DEL, or INS
  • chrom (str) – The chromosome on which the variant lies
  • genomePos (int) – The genomic position of the variant
  • ref (str) – The reference seq at the genomic position
  • obs (str) – The observed variation at the genomic position
  • coding (dict(str,MutationSyntax)) – A dictionary of associated transcripts. Key=transcript_id, value=MutationSyntax
  • isHomozygous (bool) – defines if variant is homozygous or not
  • isSynonymous (bool) – defines if variant is a synonymous mutation or not
  • offsets (dict(str,int)) – the position offset this variant has in a specific transcript-variant key=transcript-variant-id (xxx:FRED2_nn) value=offset
  • metadata (defaultdict(list)) – meta information (not relevant for core functionality of Fred2)
